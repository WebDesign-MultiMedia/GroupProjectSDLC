Customer Requirement

This outlines the requirements:
Development of an Inventory Management System (IMS).

The IMS will facilitate efficient tracking and management of inventory items for Company TG WIDGET STORE.

Planning

  • The problem is unorganized inventory and management
  • The solution is to have an inventory system/database that can scan
    and track the items
  • This is a store that sells electronics and computer hardware
  • Audience (could be anyone / everyone/ mostly techies & big companies)

Design

  • Blueprint (how will the App look)
  • Inventory list (with organized list sheet either through Excel or a POS system.)
  • Scan items (With sku code, barcode)
  • Inventory availability (Stocks)
  • Inventory Location (where items are located)
  • Restock (with automatic system when quantity is low)
  • Price Checking/Listing (Unit Price)
  • Encryption
  • newsfeed letter / weekly / biweekly

Development/ Coding

  • Application & Website
  • Programming tools (HTML, CSS, JavaScript, MongoDB, React)

Testing

  • User Experience (User Utilization)
  • Functionality Testing (App Performance)
  • Secuirty Testing (Identify & Addressing Security Vulnerabilities)
  • Load Testing (Inspect user support and level of performance)
  • Performance Testing (Speed & Efficiency of User Inputs)
  • Compability Testing (Responsive & Functional on every Operation System)
  • End User Preparation
  • Overall Quality Assurance

Deployment

  • Download through:
  • App Store
  • Google Play

Maintenance

  • Feedback of Experience
  • Report Issue or Potential Risk
  • Inspect for bugs and Syntax Error
  • Implementing Improvements & Features

Based on Everything

  • Project Duration (3 Months - 6 Months Max)
  • Two Tiers ($20-$24k for Basic Inventory App, No Features, Basic tracking, ordering, & Inventory check)
  • High Tier ($40-$45k Price checking, availability, automation, Alerts)
